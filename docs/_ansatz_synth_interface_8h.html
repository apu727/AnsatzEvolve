◆ cleanup()

int cleanup ( void * ctx)

Destroy the stateAnsatzManager created by init() and free its memory.

Parameters
ctxAddress of the pointer previously returned by init(), passed as void* (i.e. a void**). Passing a pointer to nullptr is safe and does nothing.
Return values
0Success (including the no-op case where *ctx was already nullptr).
1ctx itself is nullptr — this indicates a bug on the caller's side, not a normal "already cleaned up" state.
Note
*ctx is set to nullptr after the object is deleted, so calling cleanup() twice on the same handle is safe.

Set the angles and return the full, dense final statevector, \(\psi\) (equivalent to stateAnsatzManager::getFinalState(const std::vector<realNumType>&, vector<numType>&)).

Parameters
NAnglesNumber of angles given. May be either the compressed or decompressed count — see Angle formats in the manager documentation.
anglesThe angles to evolve to, length NAngles.
NBasisVectorsExpected length of finalState, i.e. \(2^{numQubits}\).
finalStateOutput: the dense final statevector, length NBasisVectors. Indexed such that basis state \(|XXX\rangle\) corresponds to index 0bXXX + 1, e.g. \(|100\rangle\) corresponds to 9.
ctxHandle returned by init().
Return values
0Success.
1ctx is nullptr.
2angles is nullptr.
3finalState is nullptr.
4NBasisVectors < 1.
5NAngles < 1.
6stateAnsatzManager::getFinalState() failed — see the log for details.
7The computed state's size did not match NBasisVectors — this indicates a bug on the implementation side, not a user error.
Note
If built in complex mode, this discards the imaginary part of the result and logs a warning — use getFinalStateComplex() to retrieve the full complex state.

◆ setExcitationScale()

int setExcitationScale ( int nparams,
const int * operators,
const int * orderfile,
const double * scale,
void * ctx )

Set the excitation operators and parameter ordering, with optional per-operator scale factors (equivalent to stateAnsatzManager::storeOperators() + stateAnsatzManager::storeParameterDependencies()).

Parameters
nparamsNumber of excitation operators. Must be at least 1.
operators1-indexed (Fortran-style) array of 4*nparams integers, laid out as 4 blocks of nparams values (i.e. operators[nparams*j + i] is component j of operator i), defining create/create/destroy/destroy qubit indices for each excitation. E.g. [3, 7, 4, 8] (for nparams=1) means excite from the 8th and 4th qubit into the 7th and 3rd respectively.
orderfile1-indexed array of nparams integers giving the parameter-dependency mapping (the "Order file" — see the manual). E.g. 1,1,2,3 means there are 4 operators and the first two share the same free parameter.
scaleOptional array of nparams scale factors applied to each operator's free parameter (see stateAnsatzManager::storeParameterDependencies()). Pass nullptr for a scale of 1 on every operator.
ctxHandle returned by init().
Return values
0Success.
1ctx is nullptr.
2nparams < 1.
3An entry in operators is out of bounds (qubit index exceeds 128).
4stateAnsatzManager::storeOperators() failed — see the log for details.
5An entry in orderfile is out of bounds (exceeds nparams).
6stateAnsatzManager::storeParameterDependencies() failed — see the log for details.
